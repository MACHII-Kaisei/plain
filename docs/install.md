# Plain DMG インストール手順（フォールバック）

> **推奨はソースからのビルドです**（[README](../README.md) 参照）。Widget の信頼性が高くなります。
> 本ドキュメントは Xcode を入れたくない人向けの DMG インストール手順です。

未公証のため macOS Gatekeeper の警告が出ます。以下の手順で起動してください。

## インストール

1. `Plain-x.y.z.dmg` をダウンロード
2. ダブルクリックでマウント
3. `Plain.app` を `/Applications` にドラッグ

## 初回起動

ダブルクリックではブロックされます。次のいずれかで開いてください。

### 方法A（推奨）: 右クリック → 開く

1. `/Applications/Plain.app` を **右クリック → 「開く」**
2. 警告ダイアログで **「開く」** をクリック
3. 2回目以降はダブルクリックでOK

### 方法B: ターミナルで隔離属性を外す

```bash
xattr -cr /Applications/Plain.app
```

その後ダブルクリックで起動できます。

## アンインストール

`/Applications/Plain.app` をゴミ箱に入れるだけです。
データは `~/Library/Group Containers/group.app.plain.Plain/` に残ります。完全削除する場合はこのフォルダも削除してください。
（v0.1.3 以前を使っていた場合は `~/Library/Group Containers/group.com.KaiseiMachii.Plain/` にも残っている可能性があります。新バージョン初回起動時に自動コピーされますが、不要なら手動で削除してください。）

## アップデート

v0.1.1 以降は **Sparkle による自動アップデート** に対応。
- アプリ起動時に新バージョンを自動チェック（Plain メニュー → 「アップデートを確認…」で手動チェックも可）
- 更新は同じ右クリック→開く手順で初回承認後、以降はワンクリックで適用される

## トラブルシュート

- 「開発元を検証できません」: 上記の方法A/Bで解決
- 「壊れているため開けません」: Gatekeeper の隔離が原因。方法B（`xattr -cr`）を試す
- ウィジェットが表示されない: アプリを1度起動した後、通知センター → ウィジェット編集から追加
- アップデートが届かない: メニューから「アップデートを確認…」で手動チェック。初回はネットワーク接続を確認
